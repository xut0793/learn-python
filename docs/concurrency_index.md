# 并发编程

## 计算机硬件架构的演进：单核、多核与异构

计算机硬件架构的演进核心在于不断突破物理极限，比如功耗墙（Power Wall）、内存墙（MMemory Wall ），以追求更高的计算效率和能效比。

1. 单核时代（主频为王）：早期CPU只有一个处理核心，性能提升主要依赖提高时钟频率（GHz）。单核CPU一次只能执行一条指令，多任务处理只能依靠“时间分片”技术快速切换，营造出并行的假象。其瓶颈在于主频提升会导致功耗和发热急剧增加。
1. 多核时代（核心致胜）： 为了在可控功耗下提升性能，工程师将多个独立核心集成在单个芯片上。多核CPU能够实现真正的并行处理（如四核CPU理论上可同时处理四倍指令），大幅提升了多任务处理和复杂计算（如视频渲染、3A游戏）的吞吐量。但这也带来了软件并行化的挑战，受限于阿姆达尔定律（程序的加速比受限于其串行部分的比例）。
1. 异构计算时代（专业分工）： 现代芯片演变为高度集成的SoC（片上系统）。除了CPU（分为追求极致性能的性能核P-core和追求低功耗的能效核E-core），还集成了专为图形处理的GPU、专为AI计算的NPU等。CPU的角色转变为“总指挥”，将合适的任务调度给最擅长的专用单元（如AI任务交给NPU），通过“团队协作”消除数据搬移瓶颈，实现极致的能效比。

## 计算机软件架构的抽象：程序、进程、线程与协程

计算机硬件架构的这种根本性变革，直接推动了软件层面（程序、进程、线程、协程）的演进与优化。

计算机软件架构的演进核心在于不断榨干硬件性能、不断向更轻量级、更高效率的方向抽象。

### 程序（Program）：静态的指令集合

程序是存储在磁盘上的静态文件（如 .exe 文件），包含了一系列指令和数据。它本身不占用 CPU 资源，只有当它被操作系统加载到内存并开始执行时，才会转化为动态的实体——进程。

### 进程（Process）：资源分配的基本单位

进程是静态程序的一次执行过程，也视为程序运行时的 动态实例。进程是操作系统进行资源分配基本单位，每个进程都拥有独立的内存空间（代码段、数据段、堆栈等）、文件描述符等。进程间相互隔离，稳定性高（一个崩溃不影响其他），但创建和切换的开销较大（毫秒级），在多核时代，进程间通信（IPC）的开销较大，且频繁的进程切换对 CPU 缓存极其不友好。

### 线程（Thread）：CPU调度的基本单位

随着硬件开始集成多核，为了更细粒度地利用 CPU 算力，线程应运而生。线程是进程内部的执行单元，也是操作系统进行调度的基本单位。同一个进程内的多个线程共享该进程的全部资源（如内存、文件句柄），但每个线程拥有自己独立的栈和寄存器。多核处理器的出现，使得“并行计算”成为可能。操作系统可以将一个进程中的多个线程分配到不同的物理核心上同时运行，从而成倍提升计算效率。

虽然线程比进程轻量，线程创建和上下文切换的开销远小于进程（微秒级），但它依然由操作系统内核管理（内核态）。频繁的线程切换依然涉及用户态与内核态的转换，且多线程编程面临复杂的锁竞争、死锁和数据同步问题。

### 协程（Coroutine）：用户态的极致轻量级线程

为了解决线程频繁在内核态和用户态切换，开销大、以及高并发场景下（如 Web 服务器处理数万连接）线程资源耗尽的问题，协程作为一种更极致的软件优化方案出现了。

协程是比线程更轻量的执行单位，由用户程序（通常为语言运行时），而非操作系统内核进行调度，协程的创建和上下文切换只在用户态完成，比之线程，少了内核态的切换，资源开销更低（纳秒级）。在单核或少数几个物理核心上，通过协程可以轻松地同时运行成千上万个任务（特别是在 I/O 密集型场景中）。它通过“协作式”的方式，在遇到 I/O 阻塞时主动让出 CPU，让其他协程运行，从而极大地提高了单核的 CPU 利用率。

并且协程允许开发者用同步的代码写法来实现异步的逻辑（如 async/await），规避了传统多线程复杂的锁机制。

## 计算机数据处理的方式：执行模型

数据处理方式决定了程序如何响应外部事件以及如何利用计算资源。

### 同步与异步（关注消息通知机制）：

同步： 调用方发起请求后，必须等待操作完成并得到结果后，才能继续执行后续操作（按部就班，可预测）。
异步： 调用方发起请求后立即返回，不等待结果，操作完成后通过回调、事件或通知机制告知调用方（不可预测执行顺序，但能提高响应度）。

### 阻塞与非阻塞（关注调用者的等待状态）：

阻塞： 线程在等待结果期间会被挂起（休眠），无法执行其他任务。
非阻塞： 线程在等待期间不会被挂起，可以立即返回去处理其他事情（如轮询检查状态）。

### 并发与并行（关注任务的执行关系）：

并发： 指系统具有处理多个任务的能力。在单核CPU上，通过快速的任务切换（时间分片），让多个任务交替执行，宏观上看似同时运行。
并行： 指系统在同一时刻真正同时执行多个任务，这必须依赖多核或多CPU硬件的支持。并行是并发的一种特定且理想的形式。

所以同步异步、阻塞非非阻塞、并发和并行是站在不同角度描述计算机处理数据的方式。

## 总结

总结而言， 硬件从单核走向异构提供了强大的物理基础；软件通过进程、线程到协程的层层抽象，以最小的代价榨干硬件性能；而同步/异步、阻塞/非阻塞以及并发/并行的灵活组合，则是工程师在代码层面驾驭这些资源、实现高效数据处理的根本手段。
