# 数据引用

计算在内存中存储的二进制数据，可以根据数据类型进行表示，但具体的这个值在要程序中进行使用，还需要为这个值取一个名字作表示它，就像人在社会中活动需要登记一个名字一样。

## 字面量

字面量是内置类型在代码中以常量值的表示法。

```python
None  # 空值字面量
True  # 布尔字面量
False  # 布尔字面量
10  # 整数字面量
3.14  # 浮点数字面量
0b1010  # 二进制字面量
0o10  # 八进制字面量
0x1F  # 十六进制字面量
'Hello'  # 字符串字面量
b'Hello'  # 字节串字面量
```

## 变量

计算机内存中存储的某个值，在程序代码中可能会被多次使用，所以为了方便引用，我们需要为这个值取一个名字，这个名字就叫做变量，变量的值指向这个值在内存中的内存地址。

> 名称是对数据的引用，而非数据本身。把名称想象成一个带有细绳的标签，绳子的一头绑着计算机某处内存（内存地址），一头绑着变量名称 ​。

```python
x = 42
y = 42
# 使用 id() 获取变量指向的内存地址（整数形式）。
print(id(x)) // 140727193241800
print(id(y)) // 140727193241800
```

此时你发现，x 和 y 指向的内存地址是相同的，说明它们指向的是同一个值。这个主要是因为 python 中整数的缓存机制，对于较小的整数（通常是 -5 到 256），会缓存常用的整数对象，避免重复创建相同的对象，从而提高效率。浮点数、字符串也可能因缓存机制共享地址。

如果是稍微复杂的数据类型，或者自定义类型，通常都会独立保存，产生新地址。

```python
x = [1, 2]
y = [1, 2]
print(id(x)) // 2078749635456
print(id(y)) // 2078749633408
```

## 标识符

变量的名称也称为**标识符**。通常情况下，变量的名称会使用具有描述性的单词组成，做到“见名知义”。标识符的命名规则如下：

- 标识符只能包含字母 a-z、A-Z、数字 0-9 和下划线\_。
- 标识符不能以数字开头。
- 标识符的长度没有限制，但区分大小写。
- 标识符不能是 Python 中的关键字冲突。

对于多个单词的标识符，通常使用下划线命名法（snake_case），即将单词之间用下划线连接起来，每个单词的首字母都小写。

> 下划线命名法（snake_case）：单词之间用下划线连接，每个单词的首字母都小写，比如 `btn_action`。（Python）
>
> 驼峰命名法（CamelCase）：第一个单词的首字母小写，后续单词的首字母大写，也称为小驼峰命名法，比如 `btnAction`。（Java / JavaScript）
>
> 帕斯卡命名法（PascalCase）：所有单词的首字母都大写，也称大驼峰命名法，比如 `BtnAction`。

```python
# 变量命令示例
age = 18
gender = 'male'
btn_action = 'click'
```

## 关键字

关键字，也称保留字，是指在 python 程序代码中具有特殊含义的单词，用来定义语言功能，不能作为标识符使用。

```
None        True         False
if          elif         else         while       for         in        break    continue
is          and          or           not
try         except       finally      pass        raise
def         class        return       lambda      nonlocal    global     del
async       await        yield
from        import       as
assert      with
```

还有一类标识符仅在特定上下文中被保留，它们也被称为 **软关键字**，但是通常情况下，一般也不建议使用它们作为自定义标识符。

```
match   case   type  int   float   bool    complex   list   dict   set   tuple   str   bytes   bytearray   memoryview
_  在 match 语句内部的 case 模式中，_ 是一个 软关键字，它表示 通配符。
_* 以下划线开关命名的变量，通常视为 私有变量，在语言层面不会被 `from module import * ` 导入。
__* 类的私有名称，基类及派生类的 "私有" 属性之间产生名称冲突
__*__ 语言底层系统定义的名称，通常简称为 "dunder(double under)" 魔术方法，这些名称由解释器及其实现（包括标准库）定义。
```

在终端中输入 `python` 进入交互界面，然后输入 `help('keywords')` 可以查看 python 所有的关键字。

```sh
>>> help("keywords")
```

另一种方式

```sh
>>> import keyword
>>> keyword.kwlist
```

## 常量

常量通常是指在代码执行过程中，值恒定不变的变量。同大部分语言约定一样，使用全大写字母来表示常量，有多个单词用下划线连接，比如如 MAX_SIZE、PI 等。

在 Python 中，没有内置的严格常量概念，因为无法像其他语言（如 C++、Java）那样通过语法强制变量不可修改，通常是约定成俗而已。在 Python 代码中看到全大写的变量，就是在告诉开发者 “这个变量应该被视为常量，不要修改它”。

```python

MAX_CONNECTIONS = 100
PI = 3.14159

# 技术上仍可修改，但不建议这样做
MAX_CONNECTIONS = 200  # 不会报错，但违反了约定
```

> Python 社区有一句广为流传的格言：“We are all consenting adults here”（我们都是成年人）。这句话表达了很多 Python 程序员的一个共同观点，我们应该自己对自己的行为负责而不是从语言层面来限制行为。所以在 Python 代码中有很多规范都是约定式的，比如全大写的常量，下划线开头表示的私有属性等。
