# 数据类型

计算机中的所有数据均由二进制来表达。为了表示某个内存区域中这些二进制数据的实际意义，必须要有数据类型。数据类型能够帮助我们解读二进制数据的含义，从而使我们能从待解决问题的角度来看待数据。这些內建的底层数据类型（又称原生数据类型）提供了算法开发的基本单元。

定义数据类型的作用：

- 描述了二进制数据的含义，比如说某块内存区间中的二进制数在此时是表示整数，还是字符串，还是其它数据结构。
- 描述了该数据能够参与的所有运算，比如说这块内存区间中的二进制代表整数，那它就可以进行加减乘除计算。

下面是 Python 中基本数据类型

| 数据名称 | 类型     | 示例                      |
| -------- | -------- | ------------------------- |
| None     | 空值型   | `None`                    |
| bool     | 布尔型   | `True`, `False`           |
| int      | 整型     | `123`, `-456`             |
| float    | 浮点型   | `3.14`, `-2.718`          |
| complex  | 复数型   | `3 + 4j`, `-1 - 2j`       |
| str      | 字符串型 | `'hello'`, `"world"`      |
| bytes    | 字节型   | `b'hello'`, `b'\x00\x01'` |

## None 空值

`None` 是一个特殊的空值类型，它在许多情况下它被用来表示空值，例如未显式指明返回值的函数将返回 None，在逻辑判断为假值。

注意：

- None 是一个单例对象，它不支持任何操作，包括算术运算、关系运算、逻辑运算等。
- 不能将 None 与其他数据类型进行比较，包括与整数、浮点数、字符串等。

## bool 布尔值

布尔型的有效值只有 True 和 False，表示是和否。注意这里首字母大写，区分于 Java 或 JS 中全小写形式。

- 假值 `falsy`：`None`、`''`、`False` 和 `0` 都是假值，在逻辑判断中都是 `False`。
- 真值 `truthy`：除了假值之外的所有值都是真值。

## int 整数

整数就是常规的任意数字，有正数和负数，正数的 `+` 号通常省略，负数需要在前面添加 `-` 表明。比如 `123` 或 `-456`。

另外如果一个整数数值比较大，比如 `1234567890`，那么可以使用下划线 `_` 分隔来提高可读性，比如 `1_234_567_890`，解释器会自动处理下划线 `_`，比如 `1_234_567_890` 会被解释为 `1234567890`。

### 进制

虽然代码是写给计算机执行的，但是代码是写给人看的，所以在编程中常规数值通常代表方便人类可读的十进制。比如 `123` 或 `-456`。除了十进制之外，进制类型还有常见的二进制、八进制、十六进制。

- 二进制：以 `0b` 或 `0B` 开头，后面跟着 `0` 或 `1`。比如 `0b1111011` 或 `0B1111011`。
- 八进制：以 `0o` 或 `0O` 开头，后面跟着 `0` 到 `7`。比如 `0o173` 或 `0O173`。
- 十进制：就是常规的任意数字，有正数和负数，正数的 `+` 号通常省略，负数需要在前面添加 `-` 表明。比如 `123` 或 `-456`。
- 十六进制：以 `0x` 或 `0X` 开头，后面跟着 `0` 到 `9` 或 `a` 到 `f` 或 `A` 到 `F`。比如 `0x7b` 或 `0X7B`。

```sh
# 这段二进制数值
0b1111011
# 如果表示十进制整数是
123
# 如果表示八进制整数是
0o173
# 如果表示十六进制整数是
0x7b
```

### 范围：最大值和最小值

在 Python 3 里，整数类型（int）的大小并没有固定限制，理论上它的取值范围是无限的。这意味着，只要你的系统内存足够，Python 就能够处理任意大的整数，不管这个数有多大，都不会出现溢出的情况。同理，Python 也能表示任意小的负整数。

这主要是因为 Python 的整数是动态分配内存的，其占用的内存会随着数值的增大而增加，所以能够表示极大或极小的数。但是数值越大，进行运算时消耗的时间和内存就越多。

区别于其它语言，整数都有范围限制，比如

- Java 里的整数是固定的 32 位，也就是 4 个字节，所以它的取值范围是 `-2^31` 到 `2^31 - 1`，也就是 `-2147483648` 到 `2147483647`。
- JavaScript 在底层整数和小数统一处理，以 IEEE 754 规范的双精度 64 位浮点数格式的存储，取值范围是 `-2^53` 到 `2^53 - 1`，也就是 `-9007199254740992` 到 `9007199254740991`。超出这个范围的数值可能会失去精度。

## float 小数（浮点数）

浮点型就是小数，含有小数点，比如 `3.14` 或 `-2.718`。如果一个比较大或比较小的小数，比如 `3.1415926`，形式上不好识记，就会以科学计数法的形式表示，比如

3.1415968 = 31.415968 × 10<sup>-1</sup> = 314.15968 × 10<sup>-2</sup> = 31415.968 × 10<sup>-4</sup>

这个时候，你就会发现，浮点数中的“浮点”可以理解为浮动的小数点。

> 科学计数法是一种将数字表示为 a 乘以 10 的 n 次方的形式，这种表示法使得处理非常大或非常小的数字变得更加简便。

### 存储

Python 的浮点数采用 IEEE 754 双精度标准存储，形式上也类似科学计数法，只不过它基数是 2，不是 10。在内存空间分为三部分表示：（以 64 位浮点数标准）（从右向左）

| 符号位 S                                       | 指数位 E                                      | 小数位 F                                     |
| ---------------------------------------------- | --------------------------------------------- | -------------------------------------------- |
| 占 1 位                                        | 占 11 位                                      | 占 52 位                                     |
| 第 63 位                                       | 第 62-52 位                                   | 第 51 - 0 位                                 |
| 正负数符号位（sign）<br>0 代表正数，1 代表负数 | 存储指数（exponent）<br>用来表示 2 的几次方数 | 小数位（fraction）<br>超出的部分自动进一舍零 |

浮点数存储默认整数为一位且为 1，即小数点前为 1，所以这一位不用写到内存中，节约内存空间。

根据上面的约定，一个数字的值可以根据下面的公司算出：

value = (-1)^sign × 1.fraction × 2^(exponent - 1023)

比如数字 3.5 的二进制表示为 11.1，即 1.11 × 2¹。对比上面的公式，符号位为 0（表示正数），指数位为为 1023 + 1 = 1024 (二进制表示为 10000000000)，小数位为 .11 二进制 1100...0（补零至 52 位）。

所以最终在内存中存储的二进制数为`0 10000000000 110000000000000000000000000000000000000000000000000`

## complex 复数

复数型就是包含实部和虚部的数，虚部以 j 或者 J 作为后缀。比如 `3 + 4j` 或 `-1 - 2j`。Python 内置的 complex 类型以及 cmath 模块为这些应用提供了便捷的支持。

复数通常在以下场景中广泛应用：

- 工程计算：复数在处理波动、信号以及电磁场等周期性现象时是不可或缺的工具。
- 图像处理与计算机图形学计算：比如二维变换在进行旋转、缩放等二维变换时，复数可以简化计算过程。
- 在航空航天和机器人学领域，复数可用于姿态控制：帮助计算飞行器或机器人的姿态；导航系统：辅助进行方向和位置的计算。

等等。

## str 字符串

字符串是一串字符的集合，用于表示文本信息。当然字符串中也可以表示单个字符。常规编程语言中，字符串可以使用单引号、双引号包裹来表示。比如 `'abc'` 或 `"abc"`

为什么要存在单引号和双引号两种符号呢？主要是为了方便表示字符串中的本身的引号。比如 `Let's go` 中包含了单引号，为了可以在字符串直接表示这个单引号，我们可以使用双引号包裹，比如 `"Let's go"`，不然的话，必须使用转义字符来特别标注这个单引号`'Let\'s go'`。

另外，如果使用字符串表示一段长文本，有段落，有换行，有缩进等，就需要使用三引号包裹，这样就可以保留文本的格式。比如

```python
'''
这是一个段落
    这是一个缩进段落
'''

# 或者

"""
这是一个段落
    这是一个缩进段落
"""
```

### 存储

计算机只能存储二进制数，二进制和字符之间需要一种对应关系，这就是字符集的作用。目前通用的字符集是 **unicode**。

字符集定义了文字和二进制的对应关系，为字符分配了唯一的编号，比如字母 `a` 对应的编号为 `97`，二进制表示为 `01100001`，这刚好是八位二进制数，一个字节刚好能表示。但如果是中文字符，比如 `中`，对应的编号为 `20013`，二进制表示为 `11000000 10010001`，这是 16 位二进制数，两个字节才能表示。

这个时候，就需要考虑字节序问题，到底是大端序（高位字节+低位字节：`11000000 10010001`）的形式存储，还是小端序（低位字节+高位字节: `10010001 11000000`）的形式存储呢。另外如果文本字符既有中文（两个字节），又有英文字母（一个字节），那是统一按两个字节，还是字母一个字节，汉字两个字节，如果文本还包括三个字节才能表示表情符号，又该如何编码？像这些问题都需要约定一种方案，叫做字符编码，目前通用的字符编码有**utf-8** 等。

所以字符集和字符编码不是一个概念，字符集定义了文字和二进制的对应关系，为字符分配了唯一的编号，而字符编码规定了如何将文字的编号存储到内存中（编码），以及如何从内存中读取二进制数并将其转换为字符（解码）。

> 关于字符集、字符编码更多的内容可以参考 [字符串\_字符编码](./string_char_encoding.md)

## bytes 字节串

Python 提供了 bytes 字节串类型，用来直接表达当前内存中的二进制数。在代码中，通过需要添加 `b` 前缀来表示字节串，比如 `b'123'`。它将二进制数按字节为单位（8 位，0-255 编码）以 Ascii 码（0-127）的形式显示，如果某个字节编码超过 128，就会显示为 `b'\x81\x82\x83'` 这种十六进制的形式。

bytes 类型在处理网络通信、文件读写、加密解密等要用到二进制数据的场景中非常有用。它提供了一种有效存储和操作字节数据的方式。

```python
'''
有一个包含红、绿、蓝、透明度四个通道的像素数据。我们可以直接访问字节数据的各个位置来获取每个通道的值，并进行进一步处理。
'''
pixel_data = b'\x12\x34\x56\x78'  # 假设这是一个像素的字节数据
#red = pixel_data[0]
#green = pixel_data[1]
#blue = pixel_data[2]
#alpha = pixel_data[3]
#print(f"Pixel values: red={red}, green={green}, blue={blue}, alpha={alpha}") #输出Pixel values: red=18, green=52, blue=86, alpha=120
​
# 将前三个字节的像素值从大端字节序转换为整数
rgb = int.from_bytes(pixel_data[:3], byteorder='big')
# 将第四个字节（透明度）从小端字节序转换为整数
alpha = int.from_bytes(pixel_data[3:], byteorder='little')
print(f"Pixel values: rgb={rgb}, alpha={alpha}") #输出：Pixel values: red=1193046, alpha=120
```

> 在 Python 中，字节串（bytes）和字符串（string）是两种不同类型的数据。
>
> - 字节串用于处理二进制数据，而字符串用于处理文本数据。
> - 字节串由 0-255 范围内的整数构成，而字符串由 Unicode 字符组成。
