# 模块 Module

模块本质上就是一个以 `.py` 为扩展名的 Python 源文件。它可以包含变量、函数、类以及可执行语句。

主要作用是实现独立的命名空间（Namespace）。这意味着模块内的变量、函数名不会与外部或其他模块冲突，遵循 LEGB 规则（局部 → 嵌套 → 全局 → 内置）进行变量解析。

- 标准库模块：Python 语言自带的模块（如 os, math, sys），提供基础功能。python 语言的哲学之一是"自带电池 Batteries Included"，指的是它自带了丰富且通用的标准库。Python 内置 200+ 标准库，涵盖字符处理、数据类型、数值计算、文件操作、并发执行、网络通信、多媒体、GUI、调试与开发工具、操作系统接口等领域。这种自给自足（self-sufficient）的特性，大幅减少了依赖管理的复杂度。
- 第三方模块：由社区开发，需通过 pip 或 uv 等工具安装（如 requests, numpy）。
- 自定义模块：你自己编写的 `.py` 文件。

## 导入语法

| 语法       | 示例                                                | 适用场景                                                   | 优缺点                                             |
| :--------- | :-------------------------------------------------- | :--------------------------------------------------------- | :------------------------------------------------- |
| 全量导入   | `import math`                                       | 模块功能集中，需访问多个成员                               | 优：命名空间清晰，无冲突<br>缺：调用需加前缀       |
| 指定导入   | `from math import sqrt`                             | 仅使用模块中少数几个功能                                   | 优：调用便捷，无需前缀<br>缺：可能引发命名冲突     |
| 别名导入   | `import numpy as np`                                | 模块名过长或需避免冲突                                     | 优：简化代码，业界标准（如 pd, np）                |
| 通配符导入 | `from math import *`                                | 交互式环境或临时测试                                       | 劣：强烈不推荐。破坏命名空间隔离，导致代码难以维护 |
| 相对导入   | `from . import module`<br />`from .. import module` | 这仅适用于包结构内部，且通常不能直接运行包含相对导入的脚本 |                                                    |

## 导入机制

import 语句不仅仅是简单的“复制粘贴”，它背后有一套完整的运行时流程。

### 导入的四个步骤

当你执行 import x 时，Python 内部实际上完成了以下动作：

1. 查找：在 sys.path 指定的路径列表中搜索模块或包。
1. 加载：读取源码，编译为字节码（.pyc），并执行顶层代码。
1. 缓存：将模块对象存入 sys.modules 字典中。
1. 绑定：在当前命名空间中绑定名称（如 x）。

### 模块搜索路径 (sys.path)

Python 按顺序在以下位置查找模块：

1. 当前脚本所在目录（或 **file** 的父目录）。
1. `PYTHONPATH` 环境变量指定的目录。
1. Python 标准库路径。
1. 第三方库路径（如 site-packages）。

如果遇到 `ModuleNotFoundError`，通常是因为模块不在 `sys.path` 中。你可以使用 `sys.path.append()` 动态添加路径，或者检查 `PYTHONPATH` 设置。

### 缓存机制 (sys.modules)

Python 会缓存已导入的模块。同一个模块在程序运行期间只会被加载和执行一次。后续的 import 请求会直接从缓存中获取对象，这极大地提升了性能。

## 最佳实践

1. 避免循环导入：如果模块 A 导入 B，B 又导入 A，会导致错误。解决方法通常是将导入语句移到函数内部，或者重构代码结构。
1. 使用 `if __name__ == "__main__"`:：这是一个非常重要的惯用语。它确保模块被直接运行时执行特定代码（如测试用例），而被导入时不执行。
1. 谨慎使用 reload：在开发调试（如 Jupyter Notebook）时，可用 `importlib.reload(module)` 强制重载模块。但在生产环境中应避免依赖它，因为它不会更新已存在的对象引用。
1. 显式优于隐式：推荐使用 `from package import module` 或 `import package.module`，而不是 `from package import *`，这样能保持代码的可读性和可追溯性。
1. 不要在 `__init__.py` 中做耗时操作：因为它会在每次导入包或其子模块时执行，影响启动速度。
